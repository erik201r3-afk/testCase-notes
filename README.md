# testCase-notes

Легкое круд-приложение, с некоторыми НО
1. Вместо UUID используется int id в угоду упрощения здесь и сейчас, отдал этому предпочтению прекрасно осознавая все нюансы и то что лучше использовать long id
2. Докер файл с точным эклипсом - для ускорения сборки и воспроизводимости в дальнейшем
3. Несколько копирований в докер файле - для ускорения пересборки в будущем
4. Юнит-тесты вынесены в отдельный репозиторий для удобства расширения в дальнейшем, ровно как и компоненты кода
5. Весь сервисный слой соответствует besdt practics по SOLID & DIP
6. Все кодировки HTTP-ответов соответствуют и отлично работают
